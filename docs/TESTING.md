# 测试规范与清单

本文档定义了 EaglesLab 官网的测试标准、流程和检查清单。

## 测试流程

### 1. 开发阶段

开发完成功能后，开发者应完成：

- [ ] 代码自测（本地运行 `npm run dev` 验证功能）
- [ ] 代码检查（运行 lint 工具）
- [ ] 构建测试（运行 `npm run build && npm run preview`）

### 2. 提交前检查

提交 PR 前必须完成以下检查：

- [ ] 所有功能测试通过
- [ ] 响应式测试通过（至少测试 Desktop/Mobile）
- [ ] 浏览器兼容性测试通过（至少测试 Chrome/Safari）
- [ ] 性能测试达标（Lighthouse Score ≥ 90）

### 3. 上线前检查

部署到生产环境前必须完成：

- [ ] 完整的功能测试
- [ ] 完整的兼容性测试
- [ ] 完整的性能测试
- [ ] 安全检查（无敏感信息泄露）

## 性能测试清单

### Lighthouse 测试

使用 Chrome DevTools Lighthouse 进行测试（桌面模式 + 移动模式）：

**目标指标**：

- [ ] **Performance（性能）** ≥ 90 分
- [ ] **Accessibility（可访问性）** ≥ 90 分
- [ ] **Best Practices（最佳实践）** ≥ 90 分
- [ ] **SEO（搜索引擎优化）** ≥ 90 分

**核心 Web 指标**：

- [ ] **LCP（最大内容绘制）** ≤ 2.5s
- [ ] **FID（首次输入延迟）** ≤ 100ms
- [ ] **CLS（累积布局偏移）** ≤ 0.1

### 资源加载测试

- [ ] 首屏 CSS 文件大小 ≤ 50KB（gzip 压缩后）
- [ ] 首屏 JS 文件大小 ≤ 100KB（gzip 压缩后）
- [ ] 单张图片大小 ≤ 200KB
- [ ] 字体文件总大小 ≤ 300KB
- [ ] 总页面大小 ≤ 2MB（首次加载）

### 加载速度测试

使用 Chrome DevTools Network 面板测试（Fast 3G 网络）：

- [ ] DOMContentLoaded ≤ 3s
- [ ] Load 事件触发 ≤ 5s
- [ ] 首屏可交互时间 ≤ 5s

## 可访问性测试清单

### 语义化 HTML

- [ ] 使用正确的标题层级（h1 → h2 → h3）
- [ ] 使用语义化标签（`<nav>`, `<section>`, `<article>`, `<footer>`）
- [ ] 表单字段有对应的 `<label>`
- [ ] 按钮使用 `<button>` 而非 `<div onclick="">`

### ARIA 属性

- [ ] 所有图片有 `alt` 属性
- [ ] 交互元素有 `aria-label` 或 `aria-labelledby`
- [ ] 隐藏的内容有 `aria-hidden="true"`
- [ ] 展开/收起元素有 `aria-expanded` 状态

### 键盘导航

- [ ] Tab 键可以遍历所有交互元素
- [ ] Enter/Space 键可以触发按钮和链接
- [ ] Esc 键可以关闭移动菜单/弹窗
- [ ] 焦点顺序符合逻辑
- [ ] 焦点状态清晰可见（`:focus` 样式）

### 颜色对比度

- [ ] 正文文字对比度 ≥ 4.5:1
- [ ] 大标题文字对比度 ≥ 3:1
- [ ] 按钮文字对比度 ≥ 4.5:1
- [ ] 链接文字对比度 ≥ 4.5:1

## 安全测试清单

### 敏感信息检查

- [ ] 代码中无硬编码的 API 密钥
- [ ] 代码中无硬编码的数据库密码
- [ ] `.env` 文件已加入 `.gitignore`
- [ ] 生产环境配置未提交到 Git

### XSS 防护

- [ ] 用户输入内容经过转义
- [ ] 表单提交数据经过验证
- [ ] 使用 Content Security Policy（如果配置了）

### HTTPS

- [ ] 生产环境强制使用 HTTPS
- [ ] 所有外部资源（字体、图片）使用 HTTPS

## 测试工具推荐

### 自动化测试工具

- **Lighthouse**：性能、可访问性、SEO 测试
- **WebPageTest**：详细的性能分析
- **GTmetrix**：页面速度分析
- **BrowserStack**：跨浏览器测试

### 手动测试工具

- **Chrome DevTools**：元素检查、网络监控、性能分析
- **Firefox DevTools**：可访问性检查
- **Responsively App**：响应式布局预览
- **Wave**：可访问性检查浏览器插件

### 代码质量工具

```bash
# HTML 验证
npx html-validate index.html

# CSS Lint
npx stylelint public/css/styles.css

# JavaScript Lint
npx eslint public/js/script.js
```
